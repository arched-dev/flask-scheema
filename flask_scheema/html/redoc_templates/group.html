## Grouping Data with `groupby` Grouping columns together allows for more advanced data manipulation and aggregation, similar to the SQL `GROUP BY` clause. This comes in handy when you want to summarise data in a unique way. ### Syntax To utilise grouping, include the `groupby` query parameter. You can specify just the field name, or use the table name and field name separated by a dot. - **Example Single Grouping**: `groupby=field` - **Example Multiple Grouping with Fields**: `groupby=field1,field2` - **Example Multiple Grouping with Table and Fields**: `groupby=table1.field1,table2.field2` ### Usage Imagine you are calling an API endpoint, and you want to group by the `{{ base_columns[1] }}` and `{{ base_columns[0] }}` field in the `{{ base_table }}` - Your API request would look like this: `&groupby={{ base_table }}.{{ base_columns[0] }},{{ base_table }}.{{ base_columns[1] }}` Or if you have joined `{{ relationship_table }}` with `{{ base_table }}`, and you want to group a field from each table: - Your API call might be: `&groupby={{ relationship_table }}.{{ relationship_columns[0] }},{{ base_table }}.{{ base_columns[1] }}` By employing the `groupby` query parameter, you can engage in more sophisticated data analytics, enhancing your ability to understand your dataset.
